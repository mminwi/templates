# Plans

Plan files live here. One file per task.

Filename format: `{YYYY-MM-DD}-{short-task-name}.md` (e.g., `2026-04-24-verified-customer-tag.md`).

Companion files for the same plan live alongside with suffixes:
- `{plan}-critic.md` — critic review output from Phase 2
- `{plan}-review.md` — reviewer output from Phase 4

All plan files are committed to git. They form a decision log. Do not delete old plans — mark them `Superseded` in their status field if the approach was abandoned.

See `.claude/commands/write-plan.md` for the plan file format, `CLAUDE.md` for when the Hodos pipeline applies, and `hodos.md` for the methodology handbook.
